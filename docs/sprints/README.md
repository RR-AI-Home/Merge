# Sprint Docs

Sprint documents are split by status:

- `active/` - sprint docs that drive current work.
- `completed/` - implemented or closed sprint records preserved for history.
- `canceled/` - superseded sprint records preserved as source/reference material only.

Current active sprints:

- [active/UNITY_FIRST_PLAYABLE_POLISH_SPRINT.md](./active/UNITY_FIRST_PLAYABLE_POLISH_SPRINT.md) - active Unity first playable polish and visual quality sprint.
- [active/PLATFORM_FACTORY_FOUNDATION_SPRINT.md](./active/PLATFORM_FACTORY_FOUNDATION_SPRINT.md) - active platform factory and theme compatibility sprint.

Do not use completed sprint docs as the active task list. Current work belongs in [../PROJECT_STATE.md](../PROJECT_STATE.md), [../TODO.md](../TODO.md), and the matching `active/` sprint doc.
