# Agent Call Guide

Use these prompts when routing work to a focused role.

## Platform

```text
@agent platform <task>
```

Use for shared engine behavior, theme contracts, app generation, verification gates, and standalone app identity.

## Unity Client

```text
@agent unity <task>
```

Use for Unity mobile UI, TextMeshPro, Canvas layout, merge board interactions, Unity setup, and Android build path.

## Backend

```text
@agent backend <task>
```

Use for future account, cloud save, liveops, analytics, remote config, and Java/backend contract work.

## QA

```text
@agent qa <task>
```

Use for functional verification, bug reproduction, visual QA, browser proof checks, Unity source checks, and QA documentation.

## Render Performance

```text
@agent render perf <task>
```

Use after major UI or Unity changes, especially when text blur, overlap, frame pacing, or mobile readability is a concern.

## Security

```text
@agent security <task>
```

Use when backend, auth, purchases, analytics, cloud saves, secrets, or privacy-sensitive features are introduced.

All roles should start from [SESSION_START.md](./SESSION_START.md) and [ROLES/readme.md](./ROLES/readme.md).
