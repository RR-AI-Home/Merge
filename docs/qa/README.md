# QA Reports

Detailed QA reports go here when a run needs more detail than [../QA_RUNS.md](../QA_RUNS.md).

Use this naming pattern:

```text
YYYY-MM-DD-area-short-description.md
```

Every detailed report should include:

- scope
- environment
- checks run
- findings
- bug doc updates
- follow-up risks

Confirmed active defects must also be listed in [../BUGS.md](../BUGS.md). Fixed or historical findings should stay out of the active bug list.
